# Methodology Notes

## Purpose

This repository folder supports a philosophy-of-physics article by making formal and interpretive assumptions explicit through small computational examples.

The goal is not to reduce philosophy to code. The goal is to show how computation can clarify:

- formal equivalence
- basis transformations
- probability calculation
- interpretive taxonomy
- ontological commitments
- model and claim metadata

## Core Formal Example

A normalized two-state vector:

psi = [1/sqrt(2), 1/sqrt(2)]

Born-rule probabilities:

P_i = |psi_i|^2

The same vector can be interpreted in multiple philosophical ways, including:

- ontic
- epistemic
- relational
- operational

## Key Distinction

Formal prediction and ontological interpretation are related but not identical.

A mathematical object may be:

- indispensable for calculation
- predictive in use
- interpretively contested
- representational rather than directly ontological
- physically meaningful only through invariant structure

## Computational Philosophy

The code is organized by scientific role:

- Python: quantum-state probability examples and interpretive mapping
- R: interpretive taxonomy and conceptual comparison
- Julia: two-state unitary evolution
- C++: state-vector numerical utilities
- Fortran: classic linear algebra style for normalized vectors
- SQL: structured metadata for claims and interpretations
- Rust: safe command-line probability utility
- C: low-level normalization example
